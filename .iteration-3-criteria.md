# Iteration 3 Criteria — README tone-down + manifest sync

## Context
The current `README.md` leads with marketing-tinged promises and is heavily installer-oriented. `/explore` iteration-3 (2026-04-21) established that four working principles should form the document backbone, that partial coverage should be described as mechanism rather than hedged with "(partial)", and that `.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json` `description` fields must be synced in the same change. The mechanical gates plus one qualitative deletion-test check form the acceptance surface. Strategic context: `.iteration-3/brief.md`.

## Files in scope
- `README.md`
- `.claude-plugin/plugin.json`
- `.claude-plugin/marketplace.json`
- `.harness/verifications/ac-6.*/` (authored by VER, materialized by IMP in Phase 1.5)
- `.harness/verification-registry.json` (appended by IMP in Phase 2d)

## Acceptance Criteria

### AC-6.1 — README word count ≤ 550
`wc -w README.md` returns a value no greater than 550. Code fences count toward the total.

### AC-6.2 — Hype-lexicon hits ≤ 4
Case-insensitive whole-word count of these 12 tokens in `README.md` is ≤ 4:
`powerful | seamless | revolutionary | magical | effortless | blazing | cutting-edge | game-chang | simply | just works | robust | elegant | beautiful`

### AC-6.3 — Installer block ≤ 15 lines
Any contiguous block describing how to install or run the plugin (headed "Install", "Use it", "Usage", or containing `/plugin marketplace add` / `/plugin install`) spans ≤ 15 lines total.

### AC-6.4 — Tables ≤ 1
Total markdown table count in `README.md` is ≤ 1. A table is detected by ≥ 2 consecutive lines matching `^\|.*\|$`.

### AC-6.5 — First-person count = 0
Case-insensitive count of word-boundary matches for `I`, `we`, `our`, `my`, `us` outside fenced code blocks is exactly 0 in `README.md`.

### AC-6.6 — marketplace.json description cleaned
`.claude-plugin/marketplace.json` field `plugins[0].description` does not contain the substrings "3-persona", "3-role", or "self-affirmation". Its value length is ≤ 200 chars.

### AC-6.7 — plugin.json description cleaned
`.claude-plugin/plugin.json` field `description` does not contain the substrings "3-persona", "3-role", or "self-affirmation bias". Shares at least 2 bigrams (consecutive word pairs) with marketplace.json description.

### AC-6.8 — No false hook reference
`README.md` contains no occurrence of `.claude/hooks/explore_readonly.py` or any other non-existent hook path. If a hook is mentioned, it refers to a real file present under `skills/*/`.

### AC-6.9 — Install commands are accurate
If `README.md` mentions installation, it references `/plugin marketplace add` or `/plugin install` (in-session slash commands). It does NOT contain the string `claude plugin install`.

### AC-6.10 — All 4 principles covered with honest partial-coverage structure
`README.md` contains text addressing all four principles: self-confirmation avoidance, interview-on-ambiguity, pre-arranged verification, cross-session spec persistence. Neither the token "partial" nor "(partial)" appears as a hedge adjective in `README.md`. If the text claims a hook enforces a principle, the cited hook file exists under `skills/*/`.

### AC-6.11 — Prose-matrix proxy (heading symmetry)
If `README.md` H2 count equals 4, then the standard deviation of those four H2 sections' body word counts is ≥ 25. If H2 count is ≤ 3 or ≥ 5, this AC passes trivially. This operationalizes asymmetric prose vs. parallel-matrix. The SKP qualitative deletion test is preserved as a PR-review checklist item, not automated.

## Regression discipline
